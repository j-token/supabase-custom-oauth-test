Supabase의 Custom OAuth Provider 기능을 사용하여 네이버 OAuth2를 연동합니다. Supabase가 OAuth 플로우 전체(토큰 교환, 유저 정보 조회)를 대행하므로, 클라이언트에서는 signInWithOAuth() 한 줄만 호출하면 됩니다.

1 네이버 개발자 센터 앱 등록

  1. 네이버 개발자 센터에 접속하여 로그인합니다.
  2. Application → 애플리케이션 등록을 클릭합니다.
  3. 애플리케이션 이름을 입력합니다.
  4. 사용 API에서 네이버 로그인을 선택합니다.
  5. 필요한 권한을 선택합니다:
    • 회원이름 (필수 권한 권장)
    • 이메일 (필수 권한 권장)
    • 프로필 사진 (선택)
    • 별명 (선택)
  6. 환경에서 PC웹을 선택합니다.
  7. 서비스 URL에 앱 URL을 입력합니다. (예: http://localhost:3000)
  8. Callback URL에 Supabase 콜백 URL을 입력합니다:
# 프로덕션
https://<your-project-ref>.supabase.co/auth/v1/callback

# 로컬 개발 (Supabase CLI 사용 시)
http://127.0.0.1:54321/auth/v1/callback

중요: Callback URL은 네이버 서버가 아닌 Supabase Auth 서버의 URL을 입력해야 합니다. Supabase가 토큰 교환을 처리한 뒤, 앱의 리다이렉트 URL로 다시 이동시킵니다.

등록 후 발급되는 Client ID와 Client Secret을 메모해둡니다.

2 Supabase Custom OAuth Provider 설정

방법 A: 대시보드에서 설정

  1. Supabase 대시보드에서 프로젝트를 선택합니다.
  2. Authentication → Providers로 이동합니다.
  3. New Provider → Manual configuration을 클릭합니다.
  4. 아래 정보를 입력합니다:
항목값
Identifiercustom:naver-test
NameNaver
Client ID네이버에서 발급받은 Client ID
Client Secret네이버에서 발급받은 Client Secret
Authorization URLhttps://nid.naver.com/oauth2.0/authorize
Token URLhttps://nid.naver.com/oauth2.0/token
UserInfo URLhttps://openapi.naver.com/v1/nid/me

Create and enable provider를 클릭합니다.

방법 B: API로 설정

const { data, error } = await supabase.auth.admin.customProviders.createProvider({
  provider_type: 'oauth2',
  identifier: 'custom:naver-test',
  name: 'Naver',
  client_id: 'YOUR_NAVER_CLIENT_ID',
  client_secret: 'YOUR_NAVER_CLIENT_SECRET',
  authorization_url: 'https://nid.naver.com/oauth2.0/authorize',
  token_url: 'https://nid.naver.com/oauth2.0/token',
  userinfo_url: 'https://openapi.naver.com/v1/nid/me',
  scopes: [],
});

참고: 네이버 OAuth는 scope 파라미터 없이도 앱 등록 시 선택한 권한에 따라 정보를 반환합니다. scopes 배열은 빈 배열로 두어도 무방합니다.

3 Redirect URL 허용 목록 설정

Supabase 대시보드에서 Authentication → URL Configuration으로 이동하여 앱의 콜백 URL을 허용 목록에 추가합니다.

# 로컬 개발
http://localhost:3000/auth-callback.html
http://localhost:5500/auth-callback.html

# 프로덕션
https://your-domain.com/auth-callback.html

주의: Redirect URL이 허용 목록에 없으면 인증 후 리다이렉트가 실패합니다.

4 앱에서 사용하기

옵션 A: .env 파일에 입력 (권장)

프로젝트 루트에 .env 파일을 생성하고 값을 입력합니다:

SUPABASE_URL=https://your-project-ref.supabase.co
SUPABASE_PUBLISHABLE_KEY=sb_publishable_xxxxxxxx

옵션 B: 페이지 내 입력 폼 사용

소스코드를 수정하지 않고, 로그인 페이지의 입력 폼에 직접 URL과 Key를 입력할 수 있습니다. 입력한 값은 localStorage에 저장되어 재방문 시에도 유지됩니다.

Supabase URL과 Publishable Key 확인 방법: Supabase 대시보드 → Settings → API에서 확인할 수 있습니다.

5 코드 레퍼런스

로그인 호출

// Supabase Custom OAuth로 네이버 로그인
const { data, error } = await supabase.auth.signInWithOAuth({
  provider: 'custom:naver-test',
  options: {
    redirectTo: 'https://your-app.com/auth-callback.html',
  },
});

세션 확인

const { data: { session } } = await supabase.auth.getSession();

if (session) {
  console.log('로그인됨:', session.user);
  console.log('유저 메타데이터:', session.user.user_metadata);
}

로그아웃

await supabase.auth.signOut();

인증 상태 리스너

supabase.auth.onAuthStateChange((event, session) => {
  if (event === 'SIGNED_IN') {
    console.log('로그인:', session.user);
  } else if (event === 'SIGNED_OUT') {
    console.log('로그아웃');
  }
});

트러블슈팅

"custom_provider_not_found" 오류

Supabase 대시보드에서 커스텀 프로바이더가 활성화(enabled) 상태인지 확인하세요. identifier가 custom:naver-test와 정확히 일치해야 합니다.

콜백 후 세션이 없음

  • 네이버 개발자 센터의 Callback URL이 Supabase 콜백 URL과 일치하는지 확인
  • Supabase의 Redirect URL 허용 목록에 앱 콜백 URL이 등록되어 있는지 확인
  • 브라우저 콘솔에서 에러 메시지 확인

"over_custom_provider_quota" 오류

Supabase는 프로젝트당 최대 3개의 커스텀 프로바이더를 허용합니다. 사용하지 않는 프로바이더를 삭제하거나 Supabase 지원팀에 문의하세요.
